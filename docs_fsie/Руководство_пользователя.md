# Руководство пользователя EcoMarineAI

**Аудитория:** биологи-ихтиологи, исследователи морских млекопитающих, сотрудники природоохранных организаций, научные сотрудники без опыта разработки программного обеспечения.

**Что вы научитесь делать:** устанавливать систему на свой компьютер, загружать аэрофотоснимки, получать идентификацию вида и индивидуальной особи, интерпретировать результаты, интегрировать данные в ваши исследования.

**Предварительные требования:** уверенное владение файловым менеджером и браузером. Знание командной строки желательно, но не обязательно (все основные сценарии работают через веб-интерфейс).

## 1. Назначение системы

EcoMarineAI — это библиотека и сервис с открытым исходным кодом, предназначенные для автоматической идентификации крупных морских млекопитающих (киты и дельфины) по аэрофотоснимкам. Система решает две задачи:

1. **Детекция:** определяет, присутствует ли на фотографии морское млекопитающее. Внутренний «anti-fraud gate» на базе CLIP отсеивает изображения без китов/дельфинов, чтобы не загружать дорогостоящую модель идентификации фотографиями лодок, побережий или посторонних объектов.
2. **Идентификация:** определяет вид животного (один из 30 поддерживаемых) и при возможности — конкретную особь (один из 13 837 известных модели индивидов).

Система построена на сочетании двух подходов:

- **Metric Learning** (ArcFace loss) для разделения индивидов в пространстве признаков.
- **Anti-fraud gate** на базе OpenCLIP (ViT-B/32) для фильтрации нерелевантных изображений.

**Научная новизна:** применение Vision Transformers и метрического обучения в задаче экологического мониторинга морских экосистем позволяет добиться точности ≥88% на нестандартных условиях съёмки — выше чем у классических CNN-подходов.

## 2. Установка

### 2.1 Минимальные требования

| Компонент | Требование |
|-----------|------------|
| Операционная система | Linux (Ubuntu 20.04+), macOS 11+, Windows 10+ с WSL2 |
| Оперативная память | 8 ГБ (16 ГБ рекомендуется) |
| Свободное место на диске | ~3 ГБ (ядро системы + веса моделей) |
| Установленное ПО | Docker Desktop (<https://www.docker.com/products/docker-desktop/>) |
| Интернет | Только для первичной установки и обновлений |

Python, CUDA, Git и знание IDE **не требуются**. Вся логика работы системы упакована в контейнеры Docker.

### 2.2 Первый запуск за 5 минут

Откройте терминал (в macOS — «Терминал», в Windows — PowerShell или «Командная строка» WSL) и выполните:

```bash
git clone https://github.com/0x0000dead/whales-identification.git
cd whales-identification
docker compose up --build
```

Первая сборка занимает ~2–3 минуты (Docker скачивает базовые образы, ставит зависимости Python и Node, загружает веса модели). При повторном запуске занимает ~10 секунд.

В терминале появится строка `Application startup complete` — это сигнал, что система готова. Откройте в браузере:

- **Веб-интерфейс:** <http://localhost:8080>
- **Интерактивная документация API:** <http://localhost:8000/docs>

Остановка: `Ctrl+C` в терминале или `docker compose down` из той же папки.

### 2.3 Обновление

```bash
cd whales-identification
git pull
docker compose up --build
```

Веса моделей сохраняются в именованном volume, повторно скачиваются только при изменении версии.

## 3. Работа через веб-интерфейс

### 3.1 Обработка одной фотографии

1. Откройте <http://localhost:8080>.
2. Нажмите **«1️⃣ Одиночная обработка»**.
3. Выберите файл (JPEG, PNG, любой формат, поддерживаемый браузером).
4. На экране появится превью изображения и кнопка **«Отправить»**.
5. Нажмите **«Отправить»**. Обработка занимает ~1 секунду на CPU.

Результат отображается в виде карточки одного из трёх цветов:

| Цвет | Значение | Что делать |
|:----:|----------|------------|
| 🟢 Зелёный | Обнаружено морское млекопитающее с указанием вида, индивидуального ID и уверенности модели | Использовать результат в своём анализе |
| 🟡 Жёлтый | Млекопитающее обнаружено, но уверенность ниже порога | Переснять фото или обрезать кадр ближе к животному |
| 🔴 Красный | На фото не обнаружено морских млекопитающих | Проверить что загружен правильный файл |

### 3.2 Пакетная обработка (сотни и тысячи снимков)

1. Упакуйте папку с фотографиями в ZIP-архив. В macOS: правая кнопка мыши → «Сжать». В Linux: `zip -r survey.zip photos/`. В Windows: правая кнопка → «Отправить» → «Сжатая ZIP-папка».
2. Нажмите **«2️⃣ Пакетная обработка»** в интерфейсе.
3. Выберите ZIP-файл.
4. Нажмите **«Отправить пакет»**.
5. После обработки появится диаграмма распределения видов и таблица результатов. Для каждой фотографии доступен экспорт в CSV.

Ограничение: размер ZIP-архива — до 500 МБ (настраивается через переменную окружения `MAX_UPLOAD_SIZE_MB`).

## 4. Работа через командную строку

Если вы предпочитаете писать скрипты или обрабатывать большие объёмы данных без браузера, используйте CLI:

```bash
python3 -m whales_identify predict /путь/к/фотографии.jpg
```

Вывод:

```
✓ фотография.jpg: humpback_whale (ID 1a71fbb72250, confidence 0.89, cetacean_score 0.98)
```

Флаг `--json` выдаёт полный JSON-ответ для парсинга в скриптах.

Обработка папки:

```bash
python3 -m whales_identify batch /путь/к/фотографиям/ --csv результат.csv
```

Файл `результат.csv` содержит по одной строке на каждое изображение: имя файла, вид, индивидуальный ID, уверенность, причина отклонения (если была). Открывается в Excel, LibreOffice, Google Sheets, R, Python/pandas, QGIS.

## 5. Понимание результатов

### 5.1 Поля ответа

| Поле | Описание |
|------|----------|
| `id_animal` | Название вида (например `humpback_whale`, `bottlenose_dolphin`). Всего 30 видов. |
| `class_animal` | Идентификатор конкретной особи (hex-строка длиной 12 символов). Всего 13 837 особей, известных модели. |
| `probability` | Уверенность модели идентификации (0.0–1.0). |
| `cetacean_score` | Уверенность anti-fraud gate (CLIP), что на фото действительно кит/дельфин (0.0–1.0). |
| `rejection_reason` | `null` если принято, или одна из причин отклонения (см. ниже). |
| `bbox` | Ограничивающий прямоугольник (`[x, y, width, height]`). В текущей версии совпадает со всем изображением — специализированный детектор дорсальных плавников планируется к релизу 2026 Q2. |
| `mask` | Изображение с удалённым фоном (base64 PNG), построенное библиотекой `rembg`. Используется для визуализации. |
| `model_version` | Версия модели, выдавшей предсказание — важно для воспроизводимости результатов в публикациях. |

### 5.2 Причины отклонения (`rejection_reason`)

| Значение | Причина |
|----------|---------|
| `not_a_marine_mammal` | CLIP-gate признал фото нерелевантным (лодка, берег, рыба, общий план океана без животных). |
| `low_confidence` | Gate пропустил фото, но модель идентификации выдала уверенность ниже порога. Обычно означает редкий вид или особь вне обучающего набора. |
| `corrupted_image` | Файл не удалось декодировать. Проверьте формат и целостность. |

### 5.3 Как интерпретировать уверенность

Общее правило: `probability ≥ 0.7` — надёжное совпадение, можно использовать без ручной проверки. `0.4 ≤ probability < 0.7` — рекомендуется верификация экспертом. `probability < 0.4` — отклоняется автоматически (настраивается через `min_confidence` в `whales_be_service/src/whales_be_service/configs/`).

Различие `probability` и `cetacean_score` важно: можно получить высокий `cetacean_score` (≥0.9 — точно кит) и одновременно низкий `probability` (<0.3 — какой конкретно, неизвестно). Это нормальный случай: означает новую особь, которую модель не видела при обучении. В этом случае `id_animal` (вид) всё равно достоверен — используйте его.

## 6. Частые вопросы и решения

### 6.1 «Первый запрос идёт долго»

На первом запросе сервис загружает веса моделей в память (~30 секунд на CPU). Последующие запросы обрабатываются за 0.5–1 секунду. Это штатное поведение — FastAPI lazy-loading.

### 6.2 «Сервис говорит что это не кит, хотя это точно кит»

Посмотрите значение `cetacean_score`. Если оно между 0.2 и 0.5 — фото «пограничное» с точки зрения модели. Попробуйте:

- Обрезать фото плотнее вокруг животного (CLIP-gate чувствителен к соотношению «животное/фон»).
- Улучшить освещение и разрешение.
- Снять с другого ракурса, чтобы был виден дорсальный плавник.

Если `cetacean_score` высокий (≥0.8) но `rejection_reason = low_confidence` — скорее всего, особь не входит в обучающий набор. Полагайтесь на значение `id_animal` — определение вида остаётся надёжным.

### 6.3 «Нужен доступ с другого компьютера в локальной сети»

По умолчанию сервис слушает только `localhost`. Для доступа из сети:

```bash
VITE_BACKEND=http://192.168.1.100:8000 docker compose up --build
```

Замените `192.168.1.100` на реальный IP-адрес вашего компьютера. После сборки веб-интерфейс будет доступен с любого устройства сети по адресу `http://192.168.1.100:8080`.

### 6.4 «Порт 8080 занят»

Отредактируйте `docker-compose.yml`, замените строку `"8080:80"` на, например, `"9090:80"`, сохраните файл и перезапустите `docker compose up --build`. Новый адрес веб-интерфейса — <http://localhost:9090>.

### 6.5 «Модель работает медленно»

EcoMarineAI поддерживает GPU-ускорение через CUDA. Если у вас есть NVIDIA GPU:

1. Установите [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/).
2. Добавьте в `docker-compose.yml` в секцию `backend`:
   ```yaml
   deploy:
     resources:
       reservations:
         devices:
           - driver: nvidia
             count: 1
             capabilities: [gpu]
   ```
3. Перезапустите сервисы.

Ускорение на GPU — обычно 5–10× относительно CPU.

## 7. Интеграция с базами данных и платформами мониторинга

EcoMarineAI поставляется с готовыми коннекторами:

- **HappyWhale** (`integrations/happywhale_sink/`) — публикация наблюдений в международную базу HappyWhale.
- **GBIF** (`integrations/gbif_sink.py`) — Global Biodiversity Information Facility, формат Darwin Core.
- **iNaturalist** (`integrations/inat_sink.py`) — гражданская наука, публикация наблюдений.
- **PostgreSQL** (`integrations/postgres_sink.py`) — с Alembic-миграциями для production-развертывания.
- **SQLite** (`integrations/sqlite_sink.py`) — для локального хранения на отдельном компьютере.

Полное руководство по интеграциям и примеры — в `DOCS/INTEGRATION_GUIDE.md`.

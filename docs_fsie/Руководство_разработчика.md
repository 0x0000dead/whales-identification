# Руководство разработчика EcoMarineAI

**Аудитория:** Python-разработчики, ML-инженеры, backend-разработчики, которые хотят доработать EcoMarineAI, обучить модель на своих данных, добавить новый вид/архитектуру или расширить API. Предполагается знание Python 3.11, PyTorch, FastAPI и Git.

---

## 1. Обзор архитектуры

EcoMarineAI — библиотека и веб-сервис для идентификации морских млекопитающих. Физическая структура репозитория:

```
whales-identification/
├── whales_identify/               # Core ML library (training, dataset, CLI)
├── whales_be_service/             # FastAPI backend
│   └── src/whales_be_service/
│       ├── main.py                # FastAPI app + lifespan + health/metrics
│       ├── routers.py             # /v1/* REST endpoints
│       ├── response_models.py     # Pydantic schemas
│       ├── webhooks.py            # Webhook registration service
│       ├── export.py              # Prediction history export
│       ├── inference/
│       │   ├── pipeline.py        # InferencePipeline (anti-fraud + id)
│       │   ├── anti_fraud.py      # CLIP gate (OpenCLIP ViT-B/32)
│       │   ├── identification.py  # EfficientNet-B4 ArcFace
│       │   └── ensemble.py        # Opt-in multi-stage ensemble
│       ├── models/                # Checkpoints (gitignored)
│       ├── resources/             # species_map.csv
│       └── configs/               # Calibrated thresholds
├── frontend/                      # React + TypeScript + Vite SPA
├── integrations/                  # Output sinks
│   ├── sqlite_sink.py
│   ├── postgres_sink.py           # + Alembic migrations
│   ├── happywhale_sink/           # HappyWhale API
│   ├── gbif_sink.py               # GBIF Darwin Core
│   ├── inat_sink.py               # iNaturalist API
│   ├── otel_sink.py               # OpenTelemetry traces
│   └── coordinates_api.py
├── research/
│   ├── notebooks/                 # Jupyter experiments (01–11)
│   ├── demo-ui/                   # Streamlit demo (legacy ViT)
│   └── demo-ui-mask/
├── scripts/                       # compute_metrics, download_models, etc
├── k8s/                           # Production Kubernetes manifests
├── reports/                       # Metrics JSON + Markdown
├── docs_fsie/                     # FSI deliverables (this folder)
├── DOCS/                          # Technical documentation
└── wiki_content/                  # GitHub Wiki source
```

**Ключевое правило:** вся производственная логика изолирована в `whales_be_service/`. ML-эксперименты — в `whales_identify/` и `research/notebooks/`. Конфиги (не код) — в `models_config.yaml` и `whales_be_service/src/whales_be_service/configs/`.

## 3. Пайплайн инференса

Главный класс — `InferencePipeline` в `whales_be_service/src/whales_be_service/inference/pipeline.py`. Он композирует два lazy-loaded этапа:

### Stage 1 — Anti-fraud gate

**Файл:** `inference/anti_fraud.py`

Использует OpenCLIP ViT-B/32 (`laion2b_s34b_b79k`). Вычисляет cosine similarity между эмбеддингом изображения и двумя группами текстовых промптов:
- 10 позитивных: «photo of a whale», «photo of a dolphin», «aerial photograph of a cetacean» и т.д.
- 14 негативных: «photo of a person», «street», «building», «text», «cartoon» и т.д.

`cetacean_score` = softmax-aggregated positive prompt probability. Порог отсечения калибруется и хранится в `whales_be_service/src/whales_be_service/configs/anti_fraud_threshold.yaml`. Изображения с `cetacean_score < threshold` возвращаются с `rejection_reason: "not_a_marine_mammal"`.

### Stage 2 — Identification

**Файл:** `inference/identification.py`

`IdentificationModel` — lazy-loaded singleton, загружает checkpoint через `torch.load()` при первом вызове `predict()`. Waterfall стратегия:
1. Если `efficientnet_b4_512_fold0.ckpt` присутствует — грузит EfficientNet-B4 ArcFace (production)
2. Иначе — ResNet-101 fallback
3. Иначе — ViT-L/32 (legacy, Stage-1 reproducibility)

Отчёт о текущем загруженном backend — в `.model_version` property. Значение отдается в каждом `Detection` response и в `/metrics`.

### Ensemble mode (опционально)

**Файл:** `inference/ensemble.py`

`EnsemblePipeline` — альтернативный пайплайн для high-stakes кейсов. Activate: `active_model: ensemble` в `models_config.yaml`. Стадии:
- `clip_gate` — CLIP anti-fraud (тот же класс что и в pipeline.py)
- `effb4_arcface` — идентификация
- `yolov8_bbox` — bbox refinement (stub до релиза весов YOLOv8 в HF)

Индивидуальные стадии включаются через `active_stages: [...]`. Тесты: `whales_be_service/tests/test_ensemble.py` (15 тестов со mocked stages).

## 4. Добавление новой модели

### Шаг 1. Добавить запись в `models_config.yaml`

```yaml
models:
  my_new_model:
    name: "My Custom Model"
    architecture: "MyCustomArch"  # имя класса в Python
    checkpoint: "whales_be_service/src/whales_be_service/models/my_model.ckpt"
    encoder_classes: "whales_be_service/src/whales_be_service/models/encoder.npy"
    species_map: "whales_be_service/src/whales_be_service/resources/species_map.csv"
    params:
      embedding_size: 512
      num_classes: 13837
    input_size: 512
```

### Шаг 2. Реализовать класс модели

Добавьте файл в `whales_be_service/src/whales_be_service/inference/`:

```python
# inference/my_custom_model.py
import torch, torch.nn as nn

class MyCustomArch(nn.Module):
    def __init__(self, embedding_size: int, num_classes: int) -> None:
        super().__init__()
        # ... ваша архитектура ...

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return ...
```

### Шаг 3. Зарегистрировать в фабрике

В `inference/identification.py` добавьте branch в `_load()`:

```python
if self._mode == "my_custom":
    from .my_custom_model import MyCustomArch
    self._model = MyCustomArch(...)
    self._model.load_state_dict(torch.load(self.my_custom_ckpt, map_location=self._device))
```

### Шаг 4. Обновить `models/registry.json`

Добавьте запись с SHA256 хэшем и версией:

```json
{
  "name": "my_new_model",
  "version": "1.0.0",
  "weights_path": "whales_be_service/src/whales_be_service/models/my_model.ckpt",
  "sha256": "<SHA256_hex>",
  "license": "CC-BY-NC-4.0"
}
```

Активируйте через `active_model: my_new_model` в `models_config.yaml`.

### Шаг 5. Тесты

Минимум в `whales_be_service/tests/`:
- Unit-тест forward pass с dummy input (без загрузки реального checkpoint — используйте `torch.nn.Module` mock).
- Integration test с реальным checkpoint (помеченный `@pytest.mark.slow`, запускается только в CI).

## 5. Обучение модели на своих данных

### 5.1 Подготовка датасета

Формат:

```
my_dataset/
├── images/
│   ├── 0001.jpg
│   ├── 0002.jpg
│   └── ...
└── train.csv
```

Где `train.csv`:

```csv
image,individual_id,species
0001.jpg,whale_a,humpback_whale
0002.jpg,whale_b,humpback_whale
...
```

### 5.2 Обучение через CLI

```bash
cd whales_identify
poetry run python train.py \
    --train_csv path/to/train.csv \
    --img_dir path/to/images/ \
    --model_name tf_efficientnet_b4_ns \
    --embedding_size 512 \
    --num_classes 13837 \
    --epochs 15 \
    --batch_size 32 \
    --lr 1e-4 \
    --arcface_s 30.0 \
    --arcface_m 0.50 \
    --img_size 512 \
    --output ../models/my_model.ckpt
```

Параметры:
- `--model_name` — любой backbone из `timm.list_models()`
- `--embedding_size` — размерность эмбеддинга перед ArcFace головой
- `--num_classes` — число уникальных individual_id в вашем датасете
- `--arcface_s` / `--arcface_m` — scale и margin ArcFace; для fine-tuning рекомендуется `s=30, m=0.3`; для обучения с нуля — `s=30, m=0.5`
- `--img_size` — размер входного изображения; EfficientNet-B4 оптимален на 512×512

### 5.3 Аугментации

Определены в `whales_identify/dataset.py` через Albumentations. По умолчанию:
- `A.Resize(img_size, img_size)`
- `A.HorizontalFlip(p=0.5)`
- `A.ShiftScaleRotate(shift_limit=0.1, scale_limit=0.15, rotate_limit=15, p=0.5)`
- `A.RandomBrightnessContrast(brightness_limit=0.2, contrast_limit=0.2, p=0.5)`
- `A.Normalize(mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])`
- `A.CoarseDropout(max_holes=8, max_height=32, max_width=32, p=0.3)`

Чтобы добавить свою аугментацию — отредактируйте `dataset.py::build_transforms()`.

### 5.4 Мониторинг обучения

Скрипт `train.py` пишет loss/accuracy в stdout и сохраняет checkpoints по эпохам в `--output` директорию. Для полноценного мониторинга подключите TensorBoard или Weights & Biases через свой callback.

### 5.5 Валидация

После обучения:

```bash
cd ..
python scripts/compute_metrics.py \
    --manifest data/test_split/manifest.csv \
    --update-model-card
```

Метрики сохраняются в `reports/metrics_latest.json` и автоматически инъектируются в `MODEL_CARD.md` между маркерами `<!-- metrics:start -->` / `<!-- metrics:end -->`.

## 6. Расширение API

### 6.1 Добавление нового endpoint

Редактируйте `whales_be_service/src/whales_be_service/routers.py`:

```python
@v1.post("/my-endpoint", response_model=MyResponse)
async def my_endpoint(request: MyRequest) -> MyResponse:
    """Короткий docstring."""
    # логика
    return MyResponse(...)
```

Pydantic-схемы — в `response_models.py`. Все endpoints уже покрыты middleware для rate limiting, CORS, OpenTelemetry трейсинга.

### 6.2 Webhook для push-уведомлений

EcoMarineAI поддерживает callback URL регистрацию через `POST /v1/webhook/register`. Пример:

```python
import httpx

response = httpx.post(
    "http://localhost:8000/v1/webhook/register",
    json={
        "url": "https://my-service.example.com/callback",
        "events": ["batch_completed", "prediction_rejected"],
    },
)
webhook_id = response.json()["webhook_id"]
```

При наступлении события EcoMarineAI отправляет POST запрос на зарегистрированный URL с JSON payload. Снятие: `DELETE /v1/webhook/{webhook_id}`.

### 6.3 Экспорт истории предсказаний

```bash
curl "http://localhost:8000/v1/export?format=csv&since=2026-01-01T00:00:00Z" > predictions.csv
curl "http://localhost:8000/v1/export?format=json" > predictions.json
```

Поддерживаются форматы `csv` (StreamingResponse) и `json`. Фильтр `since=<ISO8601>` опциональный.

## 9. Отладка типовых проблем

### «Первый запрос висит»

Штатное поведение lazy-loading. Модель грузится ~30 секунд на первом вызове `/v1/predict-single` после старта backend. Прогрейте вручную или увеличьте `readinessProbe.initialDelaySeconds` в k8s.

### «Модель не находит checkpoint»

Проверьте:
1. `bash scripts/download_models.sh` выполнен
2. Файл `whales_be_service/src/whales_be_service/models/efficientnet_b4_512_fold0.ckpt` существует
3. SHA256 совпадает: `shasum -a 256 whales_be_service/src/whales_be_service/models/efficientnet_b4_512_fold0.ckpt` vs `models/checksums.sha256`

### «Тесты падают на CI, но локально зелёные»

Разница окружений: CI runner использует Ubuntu, локально macOS. Частые причины:
- `libgl1` / `libglib2.0-0` не установлены — смотрите `whales_be_service/Dockerfile` для списка системных зависимостей
- Разные версии Python — проверьте `poetry env info`, должно быть `3.11.x`
- Тайминг-зависимые тесты — используйте `@pytest.mark.flaky(reruns=3)` для сетевых тестов

### «Mypy ругается на Pydantic v2»

Pydantic v2 использует `model_config` как `ConfigDict`, а не как class. Для наследования моделей используйте `model_config = ConfigDict(...)` вместо старого `class Config`. Mypy plugin pydantic обязателен — добавьте в `pyproject.toml`:

```toml
[tool.mypy]
plugins = ["pydantic.mypy"]
```

## 10. Git workflow

- Ветки: `feature/<short-desc>`, `fix/<bug-id>`, `docs/<area>`
- Commits: conventional style — `feat: ...`, `fix: ...`, `docs: ...`, `refactor: ...`, `test: ...`
- PR: минимум 1 approval, все CI checks зелёные, conflicts resolved
- Merge: squash merge в `main`
- Удаление ветки после merge: автоматически через GitHub settings

# Руководство системного администратора EcoMarineAI

**Аудитория:** DevOps-инженеры, системные администраторы, ответственные за деплой и эксплуатацию backend-сервисов. Предполагается знакомство с Docker, Linux, HTTPS/TLS и мониторингом уровня Prometheus.

**Цель документа:** обеспечить воспроизводимую установку, эксплуатацию и обслуживание production-инстанса EcoMarineAI с соблюдением целевых показателей ТЗ (доступность ≥95% за 7 дней, латентность p95 < 8 с).

---

## 1. Архитектура развертывания

EcoMarineAI состоит из трёх логических компонентов:

```
┌─────────────┐    HTTPS    ┌─────────────┐    HTTP    ┌────────────────┐
│   Клиент    │────────────▶│    Nginx    │───────────▶│  FastAPI ML    │
│ (браузер,   │             │ Ingress /   │            │  backend       │
│  CLI, API)  │             │ reverse pr. │            │ (whales_be_svc)│
└─────────────┘             └─────────────┘            └────────────────┘
                                                               │
                                                               ▼
                                                     ┌─────────────────┐
                                                     │ Model weights   │
                                                     │ PVC / local FS  │
                                                     │  - EffB4 .ckpt  │
                                                     │  - encoder.npy  │
                                                     │  - species_map  │
                                                     └─────────────────┘
```

Дополнительные компоненты:

- **Frontend** — статический bundle React, собирается в multi-stage Docker и отдаётся nginx:alpine на порту 80.
- **Prometheus scraper** — опрашивает `/metrics` бэкенда каждые 15 с (см. секцию 6).
- **Model registry** — удалённое хранилище весов моделей на HuggingFace Hub: `0x0000dead/ecomarineai-cetacean-effb4`.

## 2. Профили развертывания

Три сценария установки по возрастанию сложности и требований к инфраструктуре:

| Профиль | Время установки | Инфраструктура | Когда выбирать |
|---------|-----------------|----------------|----------------|
| **Локальный demo** | 5 минут | Docker Desktop на одной машине | Ознакомление, презентация, маленькая лаборатория |
| **Одиночный VPS** | 1 час | Ubuntu 22.04 + systemd + nginx | Стабильная нагрузка до 10 RPS, один исследовательский центр |
| **Production cluster** | 1 день | Kubernetes + HPA + Prometheus | 50+ RPS, SLA 95% за 7 дней, национальная система мониторинга |

### 2.1 Профиль «Локальный demo»

```bash
git clone https://github.com/0x0000dead/whales-identification.git
cd whales-identification
docker compose up --build
```

Доступные URL:

- Веб-интерфейс: <http://localhost:8080>
- Swagger UI: <http://localhost:8000/docs>
- Prometheus metrics: <http://localhost:8000/metrics>
- Health check: <http://localhost:8000/health>

Остановка: `docker compose down`. Логи: `docker compose logs -f backend`.

### 2.2 Профиль «Одиночный VPS» (Ubuntu 22.04)

#### Системные пакеты

```bash
sudo apt update
sudo apt install -y python3.11 python3.11-venv python3-pip \
    libgl1 libglib2.0-0 libsm6 libxext6 libxrender-dev libgomp1 \
    nginx certbot python3-certbot-nginx
```

#### Клонирование и установка зависимостей

```bash
cd /opt
sudo git clone https://github.com/0x0000dead/whales-identification.git
sudo chown -R $USER /opt/whales-identification
cd whales-identification
python3.11 -m venv .venv
source .venv/bin/activate
cd whales_be_service && pip install poetry && poetry install && cd ..
```

#### Загрузка весов моделей

```bash
bash scripts/download_models.sh
```

Скрипт:

- Скачивает веса EffB4 (~330 МБ) из HuggingFace (`0x0000dead/ecomarineai-cetacean-effb4`).
- Проверяет SHA256 каждого файла против `models/checksums.sha256`.
- Ретраит до 3 раз с экспоненциальным backoff при сетевых ошибках.
- Поддерживает `SKIP_CHECKSUMS=1` для быстрых ознакомительных установок (не рекомендуется в production).

#### Запуск через systemd

Файл `/etc/systemd/system/ecomarine.service`:

```ini
[Unit]
Description=EcoMarineAI inference API
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/whales-identification/whales_be_service
Environment="PYTHONPATH=/opt/whales-identification/whales_be_service/src"
Environment="ALLOWED_ORIGINS=https://ecomarine.example.com"
ExecStart=/opt/whales-identification/.venv/bin/python -m uvicorn \
    whales_be_service.main:app --host 127.0.0.1 --port 8000 --workers 4
Restart=on-failure
RestartSec=5
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

Активация:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now ecomarine
sudo systemctl status ecomarine
sudo journalctl -u ecomarine -f
```

#### Nginx reverse proxy с TLS

Файл `/etc/nginx/sites-available/ecomarine`:

```nginx
server {
    listen 80;
    server_name ecomarine.example.com;

    # Максимальный размер batch ZIP-архива
    client_max_body_size 500m;

    # Rate limiting: 60 запросов в минуту на IP (соответствует требованию ТЗ)
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=60r/m;
    limit_req zone=api_limit burst=20 nodelay;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Долгие batch запросы
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }

    location /metrics {
        # Metrics доступны только из приватной сети
        allow 10.0.0.0/8;
        allow 127.0.0.1;
        deny all;
        proxy_pass http://127.0.0.1:8000/metrics;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/ecomarine /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d ecomarine.example.com
```

### 2.3 Профиль «Production Kubernetes»

Манифесты находятся в папке `k8s/`:

- `k8s/deployment.yaml` — Deployment с 3 репликами по умолчанию, resources.limits (CPU 2, memory 4Gi), readiness/liveness probes.
- `k8s/service.yaml` — ClusterIP Service на порту 8000.
- `k8s/hpa.yaml` — HorizontalPodAutoscaler: min=3, max=10, target CPU 70%.
- `k8s/ingress.yaml` — Nginx Ingress с rate limit 60 req/min на IP.
- `k8s/configmap.yaml` — Environment variables (ALLOWED_ORIGINS, HF_REPO, MODEL_PATH).
- `k8s/README.md` — инструкция по установке.

Установка:

```bash
kubectl create namespace ecomarine
kubectl apply -f k8s/ -n ecomarine
kubectl rollout status deployment/ecomarine-backend -n ecomarine
```

Проверка автомасштабирования:

```bash
kubectl get hpa -n ecomarine -w
```

## 3. Переменные окружения

| Переменная | Значение по умолчанию | Назначение |
|------------|-----------------------|------------|
| `PYTHONPATH` | `src` | Расположение пакета `whales_be_service` |
| `ALLOWED_ORIGINS` | `http://localhost:5173,http://localhost:8080` | CORS whitelist, через запятую |
| `HF_REPO` | `0x0000dead/ecomarineai-cetacean-effb4` | Репозиторий весов модели |
| `MODEL_PATH` | `whales_be_service/src/whales_be_service/models/efficientnet_b4_512_fold0.ckpt` | Путь к checkpoint |
| `MAX_UPLOAD_SIZE_MB` | `500` | Лимит размера batch ZIP |
| `MIN_CONFIDENCE` | `0.10` | Порог `probability` ниже которого выдаётся `low_confidence` |
| `LOG_LEVEL` | `INFO` | `DEBUG` / `INFO` / `WARNING` / `ERROR` |
| `WORKERS` | `4` | Количество uvicorn worker-процессов |
| `RATE_LIMIT_PER_MIN` | `60` | Rate limit per IP |

Конфигурация через `.env` файл либо напрямую в systemd unit / ConfigMap.

## 4. Первичный health check

После запуска проверьте все основные эндпоинты:

```bash
# Liveness
curl http://localhost:8000/health
# Expected: {"status":"ok"}

# Readiness (модель загружена)
curl -X POST -F 'file=@data/sample.jpg' http://localhost:8000/v1/predict-single
# Expected: JSON с полем id_animal

# Metrics (Prometheus format)
curl http://localhost:8000/metrics | head -20

# Drift stats
curl http://localhost:8000/v1/drift-stats
```

Если первый `/v1/predict-single` возвращает таймаут — это штатное поведение: сервис использует lazy-loading, модель загружается при первом запросе (~30 с на CPU). Прогрейте сервис вручную после рестарта.

## 5. Мониторинг и алертинг

### 5.1 Метрики Prometheus

Endpoint `/metrics` экспортирует следующие счётчики и гейджи:

| Метрика | Тип | Назначение |
|---------|-----|------------|
| `uptime_seconds` | counter | Время работы процесса |
| `availability_percent` | gauge | (requests − errors) / requests × 100 — главная метрика SLA |
| `requests_total` | counter | Всего HTTP запросов |
| `errors_total` | counter | Запросы со статусом ≥400 |
| `predictions_total` | counter | Успешные предсказания (не rejected) |
| `rejections_total` | counter | Отклонения (anti-fraud или low_confidence) |
| `rejections_by_reason{reason=...}` | counter | Разбивка по причинам отклонения |
| `latency_avg_ms` | gauge | Средняя HTTP latency |
| `cetacean_score_avg` | gauge | Скользящее среднее CLIP positive score |

### 5.2 Prometheus scrape config

```yaml
scrape_configs:
  - job_name: ecomarine
    scrape_interval: 15s
    static_configs:
      - targets: ["ecomarine-backend:8000"]
    metrics_path: /metrics
```

### 5.3 Рекомендованные алерты

| Условие | Порог | Действие |
|---------|-------|----------|
| `availability_percent < 95` | 10 минут | Paging on-call, проверка логов и инцидентов |
| `rate(rejections_total[5m]) / rate(predictions_total[5m]) > 0.5` | 15 минут | Возможный drift данных или сбой CLIP-gate |
| `cetacean_score_avg` падение >0.1 от baseline | 1 час | Проверить смену природы входных данных (новый источник снимков) |
| `latency_avg_ms > 2000` | 5 минут | Необходимость масштабирования / GPU / profiling |

### 5.4 Grafana дашборд

В `docs/grafana/ecomarine-dashboard.json` — готовая панель с графиками availability, latency, rejection rate, throughput. Импорт: `Grafana → Dashboards → Import → Upload JSON`.

## 6. Обслуживание

### 6.1 Обновление весов моделей

```bash
cd /opt/whales-identification
git pull
bash scripts/download_models.sh  # обновит checksums.sha256 автоматически
sudo systemctl restart ecomarine
```

При смене версии модели обязательно обновите `metrics_baseline.json` для корректной работы drift detection.

### 6.2 Ротация логов

Для systemd: `sudo journalctl -u ecomarine --vacuum-time=14d` (очищает старше 14 дней).

Для Docker: logrotate не требуется, если используется `json-file` driver с `max-size` и `max-file` в daemon.json:

```json
{
  "log-driver": "json-file",
  "log-opts": {"max-size": "100m", "max-file": "5"}
}
```

### 6.3 Резервное копирование

EcoMarineAI — stateless-сервис; специальные бэкапы не требуются. Тем не менее, для воспроизводимости рекомендуется сохранять:

- `models/registry.json` — каталог обученных моделей.
- `reports/metrics_baseline.json` — baseline для drift detection.
- `whales_be_service/src/whales_be_service/configs/anti_fraud_threshold.yaml` — калиброванный порог anti-fraud gate.

Все они должны быть закоммичены в git после каждого retraining.

### 6.4 Смена модели

Процесс plug-and-play реализован через `models_config.yaml`. Чтобы переключить production-модель:

1. Добавьте новую запись в секцию `models:` с полями `architecture`, `checkpoint`, `params`, `input_size`.
2. Измените поле `active_model` на ключ новой записи.
3. Перезапустите backend. Lazy-loading автоматически подтянет новый checkpoint на первом запросе.

Откат: верните предыдущее значение `active_model` и перезапустите.

## 7. Безопасность

- **Non-root контейнер:** backend работает от пользователя `appuser` (UID 1000). Проверьте что PVC имеет соответствующие права.
- **Secrets management:** API-ключи (HappyWhale, GBIF) храните в Kubernetes Secrets или Vault. Никогда не коммитьте в git.
- **Rate limiting:** 60 req/min per IP настраивается в nginx Ingress. Дополнительно можно использовать WAF (Cloudflare, ModSecurity).
- **TLS:** только HTTPS на внешних интерфейсах. Let's Encrypt через certbot — бесплатно и автоматически.
- **Сканирование образов:** CI/CD прогоняет Trivy (`.github/workflows/ci.yml`, stage `security`). Рекомендуется дополнительно сканировать в registry перед деплоем.
- **Dependency scanning:** Dependabot + Safety для Python, `npm audit` для frontend.

## 8. Диагностика неполадок

| Симптом | Диагностика | Решение |
|---------|-------------|---------|
| 503 Service Unavailable на `/health` | `docker compose logs backend`, `kubectl describe pod` | Недостаточно памяти (нужно ≥4 ГБ), перезапустить pod |
| Первый запрос висит >60 с | Смотреть логи: `Loading model...` | Штатное lazy-load. Добавить warmup в readinessProbe |
| Высокий rejection rate | `/metrics` → `rejections_by_reason` | Проверить порог `anti_fraud_threshold.yaml`, возможный drift |
| OOM kill | `dmesg`, `kubectl get events` | Увеличить resources.limits до 6Gi, проверить `MAX_UPLOAD_SIZE_MB` |
| CUDA out of memory | `nvidia-smi` | Уменьшить batch size или переключиться на CPU временно |
| Prometheus не видит таргет | `curl backend:8000/metrics` | Проверить NetworkPolicy, firewall, namespace |
